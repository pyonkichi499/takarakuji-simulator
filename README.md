# takarakuji-simulator 🎫

宝くじの購入から抽選までを再現し、何回引けば当たるのか・収支はどうなるのかを可視化するシミュレータです。実際に手を動かして「確率」と「期待値」を体感できます。

**年末ジャンボ宝くじ**（1枚 300円）を自動で連続購入し、当選するたびに当選金額・収支・回収率がリアルタイムに更新されます。ビルド不要・依存ライブラリなしのバニラ構成なので、ファイルを開くだけ／GitHub Pages に置くだけで動きます。

## ✨ 主な機能

- **自動連続購入**：開始/停止ボタンで、くじを次々に購入する様子をリアルタイム表示
- **購入速度の切り替え**：スロー / 標準 / 高速 / 超高速 / 爆速（2 〜 200万枚/秒）
- **主要指標をリアルタイム更新**
  - 使った金額・購入枚数
  - 当選総額・当選回数
  - 収支（損益）
  - **回収率（%）**＝ 当選総額 ÷ 使った金額
- **回収率の推移グラフ**：買い続けるほど理論還元率（約 45.83%）へ収束していく様子が見える
- **当選履歴ログ**：当たった等級と金額を一覧表示。「1万円以上」「1億円以上」など**金額でフィルタ**可能
- **等級別の内訳テーブル**：各等級の当選確率・当選回数・合計金額
- **任意の予算上限**：金額を入れると使い切ったところで自動停止
- 高額当選時のハイライト演出

## 🚀 使い方

### ローカルで開く

`index.html` をブラウザでダブルクリックして開くだけです（サーバー不要）。

### GitHub Pages で公開する

このリポジトリはビルド不要の静的サイトなので、そのまま公開できます。

1. 変更を `main` ブランチに反映（マージ）する
2. GitHub のリポジトリ → **Settings** → **Pages**
3. **Source** を **Deploy from a branch** にする
4. **Branch** を `main` ／ フォルダを `/ (root)` に設定して **Save**
5. 数十秒後、`https://<ユーザー名>.github.io/takarakuji-simulator/` で公開されます

> ルートに `index.html` があるため、追加の設定やビルドは不要です。`.nojekyll` を同梱しているので Jekyll の処理も挟まりません。

## 🧮 シミュレーションの仕組み

年末ジャンボの代表的な構成（**1ユニット = 2,000万枚**）に基づく確率・賞金で抽選します。

| 等級 | 当選金 | 当選確率 |
|------|--------:|---------|
| 1等 | 7億円 | 1 / 20,000,000 |
| 1等前後賞 | 1億5000万円 | 1 / 10,000,000 |
| 1等組違い賞 | 10万円 | 1 / 100,503 |
| 2等 | 1000万円 | 1 / 6,666,667 |
| 3等 | 100万円 | 1 / 200,000 |
| 4等 | 10万円 | 1 / 10,000 |
| 5等 | 1万円 | 1 / 1,000 |
| 6等 | 3000円 | 1 / 100 |
| 7等 | 300円 | 1 / 10 |

- **理論還元率：約 45.83%**（1枚あたり期待値 約 137.5円 / 購入 300円）。実際の宝くじ（およそ 45〜46%）とほぼ同じです。
- 各くじは上記の確率で**独立に抽選**する近似モデルです（前後賞・組違い賞も、1等との連番関係を簡略化し、それぞれの確率を持つ独立の等級として扱っています）。
- 「買えば買うほど回収率は理論還元率へ収束する」一方、**高額等級は当選本数のブレが大きい**ため、序盤の回収率は大きく上下します。グラフでこの揺らぎと収束が体感できます。
